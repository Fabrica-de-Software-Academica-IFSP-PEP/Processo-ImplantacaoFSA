Processo de Desenvolvimento de Software da FSA - IFSP/PEP

A1. Preparar Ambiente de Implantação

Descrição

Instalar as bibliotecas necessárias para funcionar a aplicação e/ou assegurar que o ambiente de disponibilização da aplicação esteja conforme o esperado de modo que seja um servidor apto à disponibilizar a aplicação ou um repositório para disponibilizar a aplicação via download.

Responsáveis e responsabilidades

Scrum Team

Entrada e Saída

Entrada:

  • Acesso ao servidor e/ou repositório.

Saída:

  • Ambiente configurado e pronto para uso.
  • Bibliotecas globais instaladas e configuradas.

Tarefas

  • A1.1 Instalar bibliotecas e pacotes necessários
  • A1.2 Instalar e configurar servidor
  • A1.3 Testar a disponibilidade do ambiente

Recursos

  • Servidor físico ou virtual (nuvem/local)
    • Servidor do IFSP - PEP
    • Digital Ocean
    • AWS

A2. Disponibilizar a Aplicação para Validação

Descrição

Instalar as dependências necessárias para garantir a aplicação conforme a última versão esperada, mantendo frontend, backend e banco de dados atualizados, realizar testes de verificação para assegurar o correto funcionamento do sistema com o mínimo de erros, e por fim construir e disponibilizar a solução para acesso para homologação.

Responsáveis e responsabilidades

Scrum Team / Pipeline

Entrada e Saída

Entrada:

  • Aplicação em um estado funcional.
  • Scripts de configuração e automação.
  • Banco de dados atualizado.
  • Ambiente de homologação disponível e configurado.

Saída:

  • Sistema implantado para homologação.
  • Aplicação acessível aos usuários.

Tarefas

  • A2.1 Instalar ou atualizar aplicação
  • A2.2 Instalar e atualizar dependências
  • A2.3 Instalar ou atualizar banco de dados se necessário
  • A2.4 Executar construção (build) do sistema
  • A2.5 Configurar ambiente de homologação
  • A2.6 Realizar testes de fumaça
  • A2.7 Executar o deploy para o ambiente de homologação

Recursos

  • Ferramentas de CI/CD, Pipelines
    • GitHub Actions
    • Jenkins
  • Imagens e Contêineres para Servidores
    • Docker
  • Sistema Gerenciador de Banco de Dados
    • MySQL
    • PostgreSQL
